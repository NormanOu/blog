40个AI代理，每月500美元，顶5万美元团队：前谷歌产品负责人的AI职场革命

100倍的效率差，不是未来，而是现在 一年前，Jacob Bank还没有任何市场营销经验。 如今，他运营着40个AI代理，每月仅花费500美元，就能完成价值5万美元的营销团队的工作。 “我们都应该有几十甚至几百名AI代理为我们工作。100倍的效率差，不是未来，而是现在。” 说这话的人是前谷歌产品负责人，现在是自动化平台Relay.app的创始人兼CEO。 他不仅仅是在谈论未来——他已经活在了这个未来里。 一、从零开始：不要一口气建40个代理 Jacob的建议是：从简单开始。 “创建一个代理，它只能做一件事。创建第二个代理，它再做第二件事。然后慢慢地，你也许可以有一个代理坐在它们上面，在必要时调用它们。” 他打了个比方：这就像雇佣人类员工。你不会让一个刚入职的社交媒体人员负责所有社交媒体营销。你让他们从简单的开始。 “不要从40号代理开始，先从一个开始。” 如今，他的40个代理各司其职： 每次发布新YouTube视频，自动创建LinkedIn帖子 每次发布新视频，自动生成一条推文 每当竞争对手的CEO在推特或领英发帖，立刻通知他 每周自动查看竞争对手的定价页面，告知是否有重要变化 二、“AI是实习生”？那是错误的思维模式 Jacob承认，一年前他也犯过一个错误。 “一年前，我说过：‘嘿，把AI想象成你的实习生。‘然后我意识到，那是一种错误的思维模式。” 为什么错了？ “AI当然可以完成一类任务——那些可以节省时间的、通常你会交给实习生去做的事。但AI也能进行战略分析、竞品分析、内容创作——这些是实习生绝对无法完成的。” 他举了自己的例子。 “我不擅长打电话推销，没有销售背景。但事实证明，有些技巧在销售电话中非常有效。” 于是，他做了一个AI销售教练。 这个教练会分析他所有会议的文字记录，然后给出反馈：“你本可以更好地表达这个意思。你太容易跳到演示页面了。你应该多问一些探索性问题。” “如果你出去聘请一位销售教练，一位优秀的销售教练大约要花1万美元一个月。而你可以自己打造一个相当不错的AI销售教练，加上训练，每周大概只花五美元。” AI不是实习生，而是可以让你"超强超快"的教练。 三、未来工作：每个人都是"超级独立贡献者" Jacob在谷歌工作多年，那时有一个职位叫"技术主管经理"——一个混合型角色，既要精通技术，又要管理团队。 “这是一个非常难演的角色。技术工作和管理工作很难同时兼顾。” 但他认为，这就是我们所有人的未来。 “我们都需要成为超级独立贡献者（Super IC）。” 他这样定义未来的工作分配： “一天中三分之二的时间，我是一名个人贡献者——我在制作YouTube视频、编辑博客文章、发布内容、和顾客交谈。一天中三分之一的时间，我在协调工作——我的人工智能代理团队会帮助我处理这些。” 他做了一个大胆的预测： “大家看起来都会像我——你一天中三分之二的时间是个人贡献者的工作，三分之一的时间是协调你的AI代理团队。” 四、AI代理不是"设置完就不用管" Jacob反复强调一个重要的点：AI代理需要持续迭代。 “你不能只是构建一个人工智能代理，然后说：‘这东西对我来说肯定管用，永远。‘我一直都在修改我的人工智能代理。” 他分享了一个真实的迭代过程： 做了10次之后，他发现格式不对，客户根本没看文档。 “我应该直接把这句话写进邮件里。” 于是他"解雇"了那个代理，重新利用它做另一件事：直接把摘要放到后续邮件里。 “有时候，你不再需要某个功能了。就像一年前我们非常重视搜索引擎优化，后来我们意识到，这对我们来说并不是重要的渠道。” “所以我告诉我的AI SEO代理们：‘你们现在可以停下来了。将来我会再联系你。’” 最棒的是什么？ “**没有任何情感负担。没有分歧。没有协调成本。**我有个想法，AI助手帮我实现。他们说：‘是的，绝对的。‘那就这么办吧。” 五、“风险最大的职业，是停滞不前” Jacob对"风险"有一个颠覆性的看法。 “在我父母那一代，在一家大型财富500强公司工作40年，是一份稳定的职业。而一份充满风险的职业，要么自己创业，要么每隔几年就换一家公司。” “我认为情况已经完全颠倒了。” “**风险最高的职业，是你的技能与某一家特定公司的环境联系过于紧密。**这种情况可能会随着时间推移而改变。” “而职业生涯最稳定的，往往是那些创办自己公司的人。他们拥有许多不同的人生经历。” 他的结论很干脆： “我认为最冒险的事情，就是停滞不前。风险在于停滞。进步意味着学习、成长和挑战自我。” 六、未来两项必备技能 作为三个孩子的父亲，Jacob经常思考下一代将生活在什么样的世界里。 他总结了两项未来必备的基本技能： 第一，你表达自己想要什么、并将其告诉AI的能力如何？ 你需要清晰地表达什么才是重要的，需要做什么，然后指导AI如何做到这一点。 第二，你与他人建立社交关系的能力如何？ 人类的能力可以以AI无法做到的方式展现出来。这就是录制YouTube视频的原因——因为它体现了个性，人们由此建立起社交联系。 “我认为这些技能对于任何职业生涯的未来来说，都将是经久耐用的。” 七、面对AI，你只有两个选择 面对当前AI代理热潮，Jacob认为你只有两种反应方式： 谷歌一篇论文，存储芯片天塌了：AI内存需求暴降6倍

一篇论文，万亿市值蒸发 谁也没想到，本周三美股开盘，存储芯片板块遭遇"黑色时刻"。 美光科技下跌4%，西部数据下跌4.4%，希捷下跌5.6%，闪迪更是重挫6.5%。 引发这场抛售地震的导火索，正是谷歌发布的TurboQuant压缩算法。 Cloudflare首席执行官Matthew Prince甚至将其形容为：“谷歌的DeepSeek时刻”。 大模型的吞金兽：KV缓存 要理解TurboQuant的分量，先得搞清楚它瞄准的靶心——KV缓存到底有多吃内存。 大模型生成每一个Token时，都要"回看"之前所有Token的信息。为了避免重复计算，模型把每一层注意力机制产出的Key和Value向量全部缓存起来。 问题在于，这张"速查表"随对话长度线性膨胀。 当上下文从4K扩展到128K甚至百万级别，KV缓存吞掉的显存往往反超模型参数本身，成为推理阶段最大的内存瓶颈。 传统解法是向量量化——把16-bit浮点数压缩成4-bit整数。但几乎所有传统方法都需要为每一小块数据额外存储一组全精度的量化常数，每个数字多吃1到2个bit。 压到4-bit，实际可能是5到6-bit。压缩的意义被自己的"手续费"蚕食了一大截。 而TurboQuant的野心，正是彻底消灭这笔附加费。 两步绝杀：极坐标变换 + 1-bit误差校验 TurboQuant的核心，是一个精巧的两阶段流程。 第一阶段：PolarQuant换一个坐标系看世界 传统量化在笛卡尔坐标系（X、Y、Z轴）下操作，每个轴的取值范围不固定，必须额外存储归一化参数。 PolarQuant的第一步，是对数据向量做一次随机旋转。 这一步看似随意，背后的数学意义却很深：在高维空间里，随机旋转会让向量的每个坐标分量收敛到一种高度集中的Beta分布，而且各分量之间近似独立同分布。 不管原始数据长什么样，转完之后，统统变成"一个模子刻出来的"。 这让复杂的高维量化问题，降格为一组简单的一维标量量化问题。 然后，PolarQuant把旋转后的向量转换成极坐标系： 传统方法：向东走3个街区，再向北走4个街区 PolarQuant：朝37度方向直接走5个街区 转换之后，数据被拆成两组信息：半径（信号强度）+ 角度（信号方向）。 接下来是真正精妙的一步——递归配对。把坐标两两分组做极坐标变换，得到半径和角度；再把半径两两配对，做第二轮变换……最终整个高维向量被浓缩为一个最终半径和一系列描述性角度。 因为角度的分布模式在数学上是已知且高度集中的，整个过程不需要存储任何归一化常数。 开销，归零。 第二阶段：QJL用1-bit消灭残余误差 再精准的压缩，也会留下误差。 而且这里有个隐蔽的陷阱：一个在MSE意义上最优的1-bit量化器，在高维空间中会引入一个2/π的乘性偏差。 也就是说，你把数据压得很小、失真也很低，但用它算内积（注意力分数的核心操作）时，结果是系统性偏斜的。 TurboQuant的第二步，专门来"杀"这个偏差。 它将Johnson-Lindenstrauss变换应用到第一阶段的残余误差上，把每个误差值压缩为一个符号位：+1或-1。 配合一个特殊的估计器——用高精度的Query向量和低精度的压缩Key做联合计算。 这套组合拳，在数学上被证明是**“无偏”**的：压缩前后的内积期望值严格相等。 只消耗最后1个bit，就把第一阶段残留的系统性偏差彻底抹平。 效果有多炸裂？ 3-bit总预算，实现接近无损压缩，全程零额外开销。 在H100上，4-bit TurboQuant计算注意力logits的速度，相比32-bit基线提升了8倍。 谷歌在五大长上下文基准测试（LongBench、Needle In A Haystack、ZeroSCROLLS、RULER、L-EVal）上进行了严格验证，测试模型覆盖Gema、Mistral和Llama-3.1-8B-Instruct。 最残酷的考验来自"大海捞针"——在10万Token的文本海洋里，精准捞出一句特定信息： 4倍压缩比下，检索精度一路保持到10.4万Token，与全精度模型完全一致 6倍压缩之后，模型该记住的，一个字都没丢 一时间，全网陷入疯狂。照这样说，16GB Mac mini又能用来跑大模型了。 独立开发者在Reddit上晒出了复现成果：基于PyTorch和自定义Triton kernel，在RTX 4090上用2-bit精度跑Gemma 3 4B，输出与未压缩版本逐字符一致。 论文写的"零损耗"，社区用代码投了票。 存储芯片的天真的塌了吗？ 对于芯片存储巨头来说，这无异于一场"底层逻辑大地震"。 美光、西数等巨头的估值基石，向来建立在"AI服务器单机容量红利"之上。 一旦单次推理任务的比特需求发生结构性骤降，高性能存储的增长动能将直接面临"缩水"危机。 但存储芯片的天，大概率没有塌。 科技行业有一条反复被验证的铁律——杰文斯悖论：资源使用效率越高，总消耗量反而越大。 KV缓存压缩6倍，最可能的结果不是少买内存，而是同样的显存跑更长的上下文、更多的并发、更大的模型。 AI终于有记忆了：99%准确率刷爆SOTA，程序员的AI助手将迎来质变

AI终于不再"失忆"了 你还记得上次和ChatGPT聊了什么吗？ 它不记得。 每次打开新对话,它都是一张白纸。你告诉它的名字、偏好、项目细节,全都不见了。 这不是Bug,这是AI的"先天缺陷"——没有长期记忆。 但昨天晚上,这件事被彻底改变了。 99%准确率:AI记忆难题被攻克? 2026年3月23日,Supermemory团队扔出了一颗核弹: 超级记忆系统ASMR,在业界最难AI记忆考试LongMemEval中,刷到了99%准确率。 这是什么概念? LongMemEval不是普通的测试。它模拟的是真实生产环境中的混乱情况: 11.5万Token的对话历史(相当于一本小说) 相互矛盾的信息(你说住北京,后来又说搬去上海) 跨越多个会话的零散事件 需要时间推理的复杂问题 大多数记忆系统在这里崩溃,ASMR却拿到了99分。 全网直呼:“太疯狂了!” ASMR是如何做到的? 传统记忆系统有个致命缺陷:依赖向量数据库和嵌入(Embedding)。 简单说,就是把对话转成数字,然后通过数学计算找相似的内容。 听起来不错,但实际效果很差。为什么? 问题1:语义相似≠事实相同 “我住北京"和"我搬去上海了”,语义上很像(都在说住哪),但事实完全不同。向量搜索分不清哪个是最新的。 问题2:噪音太多 检索出100条相似信息,其中99条都是过时的,LLM根本不知道该信哪个。 ASMR的解决方案:完全抛弃向量搜索,用多Agent推理代替。 3个观察者Agent + 3个搜索Agent ASMR设计了一个"6Agent协作"的流水线: 摄取阶段(3个观察者Agent): Agent 1:读取会话1、3、5 Agent 2:读取会话2、4、6 Agent 3:读取会话7、9、11 每个Agent并行阅读原始对话,围绕六个维度提取信息: 个人信息(姓名、职业、年龄) 偏好(喜欢暗色模式、用Vim编辑器) 事件(明天有考试、下周出差) 时序数据(3月1日入职、3月15日升职) 信息更新(之前住北京→现在住上海) 助手信息(AI自己的回答历史) 检索阶段(3个搜索Agent): Agent 1:搜索直接事实和明确陈述 Agent 2:寻找相关上下文、社交线索和暗示 Agent 3:重建时间线和关系图谱 当用户提问时,这3个Agent会主动推理,而不是被动匹配关键词。 两套回答系统:8变体集群 vs 12变体决策森林 为了确保答案准确,ASMR尝试了两种方案: 方案1:8变体集群(98.6%准确率) 8个高度专业化的提示词变体并行运行 包括:精准计数器、时间专家、Context Deep Dive等 只要有一条推理路径成功,就标记为正确 方案2:12变体决策森林(97.2%准确率) 12个GPT-4o-mini驱动的Agent独立回答 引入"聚合大模型"作为最终裁判 通过多数投票、领域信任度、冲突解决机制综合答案 最终结果:ASMR在LongMemEval中达到99%准确率。 Supermemory:比ASMR更大的野心 但如果你以为ASMR只是一个刷榜实验,那就太小看这个团队了。 ASMR背后,是一个叫Supermemory的完整记忆引擎——让AI真正拥有记忆的基础设施。 ...

AI演员遭抵制，我们到底在害怕什么？

最近，“AI演员"的话题在抖音上引发热议，热度高达900多万。很多人在问：AI到底能不能取代演员？ 一个程序员视角的思考 作为一个程序员，我对AI的态度一直是：拥抱变化，但保持警惕。 AI能演戏吗？技术上已经可以了。深度伪造（Deepfake）技术已经能把一个人的脸"贴"到另一个人的身上，甚至可以生成完全虚拟的数字人。从技术角度看，AI演员不是"能不能"的问题，而是"什么时候"的问题。 但为什么这么多人抵制？ 我们害怕的不是技术，是"虚假” 人类对"真实"有近乎执念的追求。我们看电影，表面上是在看故事，实际上是在感受另一个真实的人的情感表达。 一个演员哭了，我们被打动，是因为我们知道： 这是一个真实的人 他/她经历过真实的情感 这份情感通过表演传递给我们 AI可以模拟眼泪，但无法模拟"为什么会流泪"。 AI演员的真正威胁 我认为AI演员最大的威胁不是"取代演员"，而是让整个行业变得更加工业化、流水线化。 当AI可以批量生成"及格线"以上的表演时： 底层演员的生存空间会被挤压 中层演员需要更加努力证明自己的"不可替代性" 只有顶层演员的"灵魂表演"才能站稳脚跟 这和程序员行业很像——AI可以写代码，但很难写出"优雅的架构"。 我的判断 AI演员不会完全取代人类演员，但会： 改变行业标准：观众对表演的要求会更高 重塑行业分工：更多演员转型为"表演设计师"或"情感顾问" 创造新的可能性：比如让已故演员"重返银幕" 技术从来不是中立的，它放大的是使用者的意图。 关键问题是：我们想用AI讲什么样的故事？ 你怎么看AI演员？欢迎在评论区讨论。
